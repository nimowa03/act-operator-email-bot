배경지식

먼저, AI 에이전트가 뭔가요?

🤖
일반 AI (챗봇)
질문하면 답만 해줌
"서울 날씨 알려줘" → "맑음, 22도"

= 대화만 가능
🦾
AI 에이전트
스스로 판단하고 행동함
"비 오면 우산 주문해줘" → 날씨 확인 → 쿠팡 주문

= 도구를 사용해서 실제 행동
💡

AI 에이전트는 "생각 → 도구 사용 → 결과 확인 → 다시 생각"을 반복합니다. 이걸 ReAct 패턴이라 부릅니다.

Problem
🚨

근데 AI가 마음대로 행동하면?

"김팀장에게 회의록 보내줘" → AI가 즉시 전송

→ 잘못된 주소로 발송됨 → 기밀 유출 😱
→ 되돌릴 수 없음
🔑

해결책: 민감한 행동 전에는 반드시 사람이 확인하고 승인해야 한다 → 이것이 Human-in-the-Loop (HITL)

배경지식

LangGraph가 뭔가요?

AI 워크플로우를 그래프로 만드는 프레임워크
지하철 노선도처럼,
AI의 행동을 노드(역)와 엣지(노선)로 설계

[입력] → [AI 판단] → [도구 실행] → [출력]
왜 그래프인가?
분기 — 조건에 따라 다른 경로
중단 — 특정 지점에서 멈춤 (HITL!)
반복 — 결과가 부족하면 다시 시도

일직선 코드로는 이런 흐름을 만들기 어려움
🚇

비유: LangGraph = AI의 지하철 노선도. 각 역(노드)에서 할 일을 정하고, 노선(엣지)으로 연결합니다. 특정 역에서 하차(interrupt)할 수도 있습니다.

배경지식

Act Operator = 프로젝트 자동 생성기

LangGraph 프로젝트를 밑바닥부터 만드는 건 번거롭다.
폴더 구조, 상태 관리, 도구 연동, 테스트...

Act Operator는 이 모든 뼈대를 명령어 한 줄로 생성해준다.
01
act new
프로젝트 생성
→
02
uv sync
의존성 설치
→
03
코드 구현
Claude Code
→
04
langgraph dev
시각화 + 테스트
🎬

Act = 영화 한 편 (전체 프로젝트)  |  Cast = 배우 (개별 워크플로우)
하나의 Act 안에 여러 Cast(챗봇, 이메일봇, 분류기 등)를 독립적으로 관리합니다.

라이브 시연 1

터미널에서 실제 실행

# Step 1: 이메일 작성 요청
graph.invoke({"query": "테스트 이메일 보내줘"})

⏸️ === 이메일 승인 요청 ===
수신자: makegrowth101@gmail.com
제목: 테스트 이메일
본문: 안녕하세요, 테스트입니다...
========================
승인하려면 'yes', 거부하려면 'no'

# ⏸️ 여기서 멈춤! 사용자가 결정할 때까지 대기

# Step 2: 사용자가 승인
graph.invoke(Command(resume="yes"))

✅ 이메일 전송 완료!
