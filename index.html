배경지식 1

AI 에이전트란 무엇인가?

🤖
일반 AI (챗봇)
질문하면 답만 해줌
"서울 날씨 알려줘" → "맑음, 22도"

대화만 가능하고, 직접 행동하지 않음
🦾
AI 에이전트
스스로 판단하고 행동함
"비 오면 우산 주문해줘" → 날씨 확인 → 쿠팡 주문

도구를 사용해서 실제로 행동함
💡

AI 에이전트는 "생각 → 도구 사용 → 결과 확인"을 반복합니다. 단순히 대답만 하는 챗봇과 달리, 이메일 전송, 검색, 결제 같은 실제 행동을 수행할 수 있습니다.

배경지식 2

근데 AI가 마음대로 행동하면?

"김팀장에게 회의록 보내줘"
→ AI가 확인도 없이 바로 전송
→ 잘못된 주소로 발송됨 → 기밀 유출 😱
→ 되돌릴 수 없음
🔑

해결책: 민감한 행동을 하기 전에 반드시 사람이 확인하고 승인해야 한다.
이것이 Human-in-the-Loop (사람이 중간에 확인하는 구조) 입니다.

배경지식 3

LangGraph — AI 워크플로우를 그래프로 설계하는 도구

무엇인가?
AI의 작업 순서를 설계하는 도구

지하철 노선도처럼
노드(역) = 각 작업 단계
엣지(노선) = 작업 순서 연결
왜 그래프인가?
분기 — 조건에 따라 다른 경로로
중단 — 특정 지점에서 멈춤 (HITL)
반복 — 결과가 부족하면 다시 시도

일직선 코드로는 이런 흐름이 어려움
🚇

LangGraph = AI의 작업 흐름표. 각 역(노드)에서 할 일을 정하고, 노선(엣지)으로 연결합니다. 특정 역에서 하차(중단)해서 사람의 확인을 받을 수도 있습니다.

배경지식 4

Act Operator — LangGraph 프로젝트 자동 생성기

LangGraph 프로젝트를 처음부터 만들면 폴더 구조, 상태 관리, 도구 연동 등을 직접 설정해야 한다.
Act Operator는 이 뼈대를 명령어 하나로 생성해준다.
명령어하는 일비유
act new프로젝트 폴더와 파일을 자동 생성건물의 골조 세우기
uv sync필요한 라이브러리 설치건축 자재 반입
act cast새 워크플로우 추가새 층 추가
langgraph dev개발 서버 실행 + 시각화준공 검사
🎬

Act = 전체 프로젝트  |  Cast = 개별 워크플로우
하나의 Act 안에 여러 Cast (챗봇, 이메일봇, 분류기 등)를 독립적으로 관리합니다.

코드에서 interrupt() — 실행을 멈추는 함수 — 를 호출하면 그래프가 멈춥니다.
사용자가 resume (다시 시작) 명령을 보내면 멈춘 곳에서 이어서 실행됩니다.
Checkpointer (멈춘 상태를 저장하는 장치) 가 있어야 다시 이어갈 수 있습니다.

라이브 시연

실제로 이메일을 보내봅시다

# 1단계: 이메일 작성 요청
"makegrowth101@gmail.com에게 테스트 메일 보내줘"

# 2단계: AI가 초안을 작성하고... 멈춤!
⏸️ === 이메일 승인 요청 ===
수신자: makegrowth101@gmail.com
제목: 테스트 이메일
본문: 안녕하세요, 테스트입니다...
승인하려면 'yes', 거부하려면 'no'

# 3단계: 사람이 승인하면 → 진짜 전송!
✅ 이메일 전송 완료!
👉

지금부터 터미널에서 실제로 실행합니다. AI가 초안을 만들고, 멈추는 순간을 직접 보세요.
